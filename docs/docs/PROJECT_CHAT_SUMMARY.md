## 从历史会话继承的关键经验

- 领星全局配置倾向于“全局可读、项目复用”，不要只把规则留在聊天里。
- 正式领星客户端和全局配置比临时脚本更可靠；调试时优先复用已验证路径。
- LingXing MCP 曾验证为可用，后续应优先通过 MCP 或正式客户端做能力确认。
- 领星官方文档访问、鉴权、签名、限速规则已经写入本机全局文档；需要最新细节时读取本机文档并按官方文档核验。
- Listing/ERP 页面任务可结合浏览器自动化读取实时页面，但最终应沉淀为可复跑的数据缓存或脚本。
- 周报类报告应区分“已覆盖”“未覆盖”“周年提醒”等业务状态，不要用宽泛模糊匹配替代用户指定的内部分类规则。

1. 读取根目录 `AGENTS.md`。

4. 执行任务时优先做可复跑脚本、明确输出路径和验证结果。
